# Analysis Rules 分析规则

## Part 1: Pattern Recognition 模式识别

### 数据获取

从 Notion 交易记录数据库读取数据，默认分析最近 50 笔已平仓交易。用户可指定时间范围或笔数。

### 分析维度

#### 1. 策略表现分析
```
对每个策略，计算：
- 交易笔数
- 胜率 = 盈利笔数 / 总笔数
- 总P&L
- 平均盈利 / 平均亏损
- 盈亏比 = |平均盈利| / |平均亏损|
- 期望值 = (胜率 × 平均盈利) - ((1-胜率) × |平均亏损|)
```

#### 2. 品种表现分析
同策略分析的计算，按品种维度。

#### 3. 交易周期分析
同上，按交易周期（5分钟 / 15分钟 / 1小时 / 1日）维度。

#### 4. 方向分析
对比做多 vs 做空的胜率和P&L。

#### 5. 时间段分析
如果有精确到小时的开仓时间：
- 早盘 (9:30-10:30)
- 中盘 (10:30-14:00)
- 尾盘 (14:00-16:00)

#### 6. 错误频率与成本
```
对每种错误类型，计算：
- 发生次数
- 造成的总亏损
- 占总亏损的百分比
- 最近一次发生时间
```

#### 7. 连续盈亏检测
```
扫描交易序列，识别：
- 最长连续盈利 streak
- 最长连续亏损 streak
- 当前streak状态
- 亏损后下一笔交易的胜率（revenge trading 检测）
```

#### 8. R-Multiple 追踪
```
R = 实际盈亏 / 初始风险
如果用户没有记录初始风险（止损位），使用入场价的2%作为默认R值。
计算：
- 平均R
- 期望R = (胜率 × 平均正R) + ((1-胜率) × 平均负R)
- 最大R (最佳交易)
- 最小R (最差交易)
```

|------|------|------|-------|-------|-------|
| ... | ... | ... | ... | ... | ... |

### ⚠️ 错误分析
| 错误类型 | 次数 | 总亏损 | 占比 |
|---------|------|-------|------|

### 分析阈值与信号

| 指标 | 绿灯 | 黄灯 | 红灯 |
|------|------|------|------|
| 胜率 | > 50% | 40-50% | < 40% |
| 盈亏比 | > 2.0 | 1.0-2.0 | < 1.0 |
| 期望值 | > 0 | = 0 | < 0 |
| FOMO频率 | < 10%交易 | 10-20% | > 20% |
| 复盘完成率 | > 80% | 50-80% | < 50% |

## Part 2: Streak 激励系统

### Streak 计算逻辑

```
1. 从复盘日历获取所有 类型=日复盘 的记录
2. 提取日期列表，按日期降序排列
3. 从今天（或最近一个交易日）开始往回数
4. 跳过周末（周六日）和美股休市日
5. 连续有复盘记录的交易日数量 = streak
```

### 美股休市日（2026年参考）

- 1月1日 New Year's Day
- 1月19日 MLK Day
- 2月16日 Presidents' Day
- 4月3日 Good Friday
- 5月25日 Memorial Day
- 6月19日 Juneteenth
- 7月3日 Independence Day (observed)
- 9月7日 Labor Day
- 11月26日 Thanksgiving
- 12月25日 Christmas

> **注意**: 每年休市日不同，执行时应 web search "NYSE holiday schedule [year]" 确认。

### Streak 消息模板

#### 每次复盘后显示
```
🔥 复盘Streak: {N}天
📊 本月复盘率: {completed}/{total_trading_days} ({percent}%)
```

#### 里程碑消息

| 天数 | 消息 |
|-----|------|
| 7 | 🎯 **一周达成！** 复盘习惯正在形成。Market Wizards 的交易员都坚持做复盘——你在正确的路上。 |
| 14 | 🔥 **两周不断！** 研究表明21天形成习惯，你已经走了2/3。 |
| 21 | 💪 **21天！习惯养成！** 从今天起复盘已经是你交易系统的一部分了。 |
| 30 | 🏆 **一个月！** 30天连续复盘，你已经超越了95%的散户交易者。 |
| 50 | ⭐ **50天传奇！** 半个世纪的日复盘。你的交易模式识别能力一定已经提升了。 |
| 100 | 🏅 **百日复盘大师！** 100天的数据积累让你的pattern recognition进入全新层次。 |
| 365 | 👑 **一整年！** 你做到了绝大多数人做不到的事。这份纪律就是你的edge。 |

#### 断链提醒

如果昨天是交易日但没有复盘记录：
```
⚠️ 你的 {previous_streak} 天复盘streak在昨天断了。
没关系——最重要的是重新开始。今天复盘一下？

💡 提示：哪怕写一句"今天没交易，观望"也算复盘。保持记录的习惯比内容完美更重要。
```

#### Streak 断链后重启
```
🔄 新的streak开始了！Day 1。
上一次streak: {last_streak}天
历史最长: {best_streak}天
目标：超越历史最长！
```

## Part 3: Weekly Playbook 周度策略报告

### 触发时机

- **建议**: 每周日 7:00 PM EST 运行
- **手动**: 用户说"周报"/"weekly playbook" 或运行 `/project:weekly-playbook`

### 数据收集

1. 从 Notion 获取本周一至今（或上周一至上周五）的所有交易记录
2. 从复盘日历获取本周的复盘记录
3. Web search 下周重大事件日历

### 对比上周的计算

需要分别获取本周和上周的交易记录，计算差值：
- 胜率变化用百分点(pp)
- P&L变化用绝对值
- 错误次数用箭头表示趋势

### 下周关注事件获取

使用 web search 搜索：
1. "economic calendar next week [date]"
2. "earnings calendar next week"
3. 与用户持仓相关的特定事件

### 写入 Notion

报告完成后，自动写入复盘日历：
- 类型: 周复盘
- 日期: 本周日
- 关联本周所有交易记录
- 报告内容写入页面正文

## Part 4: Pre-Market Brief 盘前简报

### 内容模板

```markdown
# ☀️ 盘前简报 | [日期]

## 持仓状态
| Ticker | 方向 | 数量 | 入场价 | 当前价 | 浮盈 |
|--------|------|------|-------|-------|------|
| ... | | | | [需web search] | |

## 今日到期
[如有到期的期权或计划平仓的交易]

## ⚠️ 注意事项
基于你的历史交易pattern：
- [如：连续盈利3天，注意不要过度自信]
- [如：上次FOMO发生在类似市场条件下]
- [如：本周已交易N笔，考虑是否需要减少频率]

## ✅ Pre-Trade Checklist
- [ ] 市场整体方向判断？
- [ ] 今天有没有重大数据/事件？
- [ ] 止损位确定了吗？
- [ ] 仓位大小合理吗？
- [ ] 这笔交易符合你的策略框架吗？
```

### 持仓当前价获取

通过 web search 获取持仓中交易的当前价格，计算浮盈。

### Pattern-Based 注意事项

根据历史数据生成个性化提醒：
- 如果当前处于连续盈利streak → 提醒谨慎
- 如果昨天有亏损 → 提醒避免revenge trading
- 如果本周交易频率异常高 → 提醒控制节奏
- 如果最近某个错误类型频繁出现 → 专门提醒
