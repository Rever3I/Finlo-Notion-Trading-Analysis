# 交易模式分析

执行 Pattern Recognition 分析报告。

## 前置检查

1. 读取 `trading-config.md` 确认已完成配置
2. 读取 `references/analysis-rules.md` 获取完整分析规则

## 数据获取

使用 Notion MCP 工具从交易记录数据库读取数据，默认分析最近 50 笔已平仓交易。用户可指定时间范围或笔数。

## 分析维度

1. **策略表现** — 每个策略的胜率、总P&L、盈亏比、期望值
2. **品种表现** — 按品种维度统计
3. **交易周期** — 按周期维度统计
4. **方向分析** — 做多 vs 做空的胜率和P&L
5. **时间段分析** — 早盘/中盘/尾盘表现
6. **错误频率与成本** — 每种错误类型的次数和造成的亏损
7. **连续盈亏检测** — streak 检测 + revenge trading 识别
8. **R-Multiple 追踪** — 平均R、期望R、最大/最小R

## 输出格式

```markdown
## 交易模式分析报告

### 核心发现
1. [最重要的 pattern，附数据]
2. [第二重要的 pattern，附数据]
3. [第三重要的 pattern，附数据]

### 策略表现
| 策略 | 笔数 | 胜率 | 总P&L | 盈亏比 | 期望值 |

### 错误分析
| 错误类型 | 次数 | 总亏损 | 占比 |

### 改进建议
1. [具体可执行的建议]
2. [具体可执行的建议]
3. [具体可执行的建议]
```

## 分析阈值

参考 `references/analysis-rules.md` 中的绿灯/黄灯/红灯阈值判断各指标健康度。
